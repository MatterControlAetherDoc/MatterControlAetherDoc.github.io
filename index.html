Element Documentation

Both MatterControl and MatterSlice have been modified to adapt to the Aether bioprinter.

NOTE: Currently all main development is happening in the 1.7 branch of the Element git repository and its submodules (MatterSlice and agg-sharp). Make temporary/personal branches off the 1.7 branch. The master branch is to be kept around to see commit order if needed. Not that it matters, but the active development branch in upstream MatterControl is design_tools.

Element (Formerly MatterControl)

MatterControl was extended to allow specification of multiple tool types and various individual tool settings for each type. This tool setting specification is done in the same application that the 3D model is sliced in. Implementation details are located here.

MatterSlice

MatterSlice is the default slicing engine used by MatterControl. This version of MatterSlice has been modified to adapt the slicing algorithm to adjust the line spacing throughout the slicing process, override general slicer speed settings, and insert tool appropriate code during tool change and extrusion/non-extrusion moves. Implementation details are located here.

Creation Plugins

Plugins can be easily added to Element to allow for arbitrary, easy expansion of element features.

UV Path Generator

This is a plugin that was created to aid in the use of a curing LED on Aether printers. The mesh viewing and interaction (e.g., click, drag, delete, etc.) is largely copied from MatterControl's TextCreator plugin. The main class of this plugin is View3DPathGenerator which is heavily based on the View3DTextCreator class. Both of these classes are subclasses of PartPreview3DWidget.

The UVPathMeshMaker class is used to do the actual path mesh generation. The entire UV path is generated from a private PathStorage instance. Each segment in the UP path is added to the path storage as a clockwise listing of the four corner veracities of the segment. When this data is converted into a mesh, all the segment's intersections are merged to form one solid path. This, as a result, generates one contiguous, manifold mesh.
